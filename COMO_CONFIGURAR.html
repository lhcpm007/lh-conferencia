🚛 LH Conferência de Cargas — Guia de Configuração

Este guia explica como configurar o Firebase (banco de dados) e o EmailJS (envio de e-mail).

Importante: As configurações são feitas no arquivo js/config.js. Após alterar, salve o arquivo e atualize o aplicativo no celular.

1. Firebase Firestore (banco de dados compartilhado)

O Firebase permite que todos os celulares acessem os mesmos dados de coletas e carregamentos.

Passo 1 — Acesse console.firebase.google.com
Passo 2 — Clique em Criar projeto → nomeie como lh-conferencia → desative Google Analytics → Criar
Passo 3 — No menu lateral: Firestore Database → Criar banco de dados → Modo de produção → Selecionar localização southamerica-east1
Passo 4 — No menu lateral: Regras do Firestore → Cole as regras abaixo e clique em Publicar:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
Passo 5 — No menu lateral: ⚙️ Configurações do Projeto → Suas apps → Adicionar app → Ícone Web (</>)
Passo 6 — Nomeie a app como lh-conferencia-web → Registrar app → Copie o objeto firebaseConfig
Passo 7 — Abra o arquivo js/config.js e substitua os valores da seção firebase:
firebase: {
  apiKey:            "AIzaSy...",           // ← cole aqui
  authDomain:        "lh-conferencia.firebaseapp.com",
  projectId:         "lh-conferencia",
  storageBucket:     "lh-conferencia.appspot.com",
  messagingSenderId: "123456789",
  appId:             "1:123456789:web:abc..."
},

2. EmailJS (envio automático de e-mail)

O EmailJS envia e-mails diretamente do navegador, sem necessidade de servidor.

Passo 1 — Crie uma conta gratuita em emailjs.com
Passo 2 — Menu Email Services → Add New Service → Gmail → conecte a conta de e-mail
Passo 3 — Menu Email Templates → Create New Template. Use o conteúdo abaixo:
CampoValor
SubjectLH Conferência — {{tipo}} — {{placa}} — {{data}}
Body (HTML)Ver abaixo
To email{{to_email}}

Conteúdo do corpo do e-mail (HTML):

<h2>LH Conferência de Cargas</h2>
<p><strong>Operador:</strong> {{operador}}<br>
<strong>Processo:</strong> {{tipo}}<br>
<strong>Placa:</strong> {{placa}}<br>
<strong>Data:</strong> {{data}}</p>
<p>A planilha Excel foi gerada e está disponível no dispositivo do operador.</p>
Passo 4 — Salve o template e anote o Template ID (ex: template_abc123)
Passo 5 — Menu Account → copie a Public Key
Passo 6 — Abra js/config.js e preencha:
emailjs: {
  serviceId:  "service_abc123",   // ← Service ID
  templateId: "template_abc123",  // ← Template ID
  publicKey:  "user_XXXXXXXXXXX"  // ← Public Key
},

3. Hospedagem no GitHub Pages

Passo 1 — Crie uma conta em github.com
Passo 2 — Crie um novo repositório: New repository → nome lh-conferencia → Public → Create
Passo 3 — Faça upload de todos os arquivos da pasta carga-conferencia usando o botão uploading an existing file
Passo 4 — No repositório: Settings → Pages → Source: Deploy from a branch → Branch: main → /root → Save
Passo 5 — Aguarde ~2 minutos. A URL será: https://SEU_USUARIO.github.io/lh-conferencia
⚠ Câmera requer HTTPS! O GitHub Pages já fornece HTTPS automaticamente. Se quiser testar localmente, use o VS Code com a extensão "Live Server" ou rode:
npx serve . na pasta do projeto.

4. Adicionar à tela inicial do celular

O site funciona como um aplicativo instalável:

  • Android (Chrome): Menu (⋮) → Adicionar à tela inicial
  • iPhone (Safari): Botão Compartilhar → Adicionar à Tela de Início

5. Alterar a senha de liberação

A senha padrão é LHADMIN1610.. Para alterar:

1. Abra o browser e pressione F12 → Console
2. Digite: btoa('NOVA_SENHA') e pressione Enter. Copie o resultado.
3. Em js/config.js, substitua o valor de _ap pelo resultado copiado.
